# DaggerConceptApp

> A step-by-step Android reference app covering all core Dagger 2 concepts — from basic injection to scopes, components, and subcomponents.
> Language: Kotlin | DI: Dagger 2

---

## About

This repository is a focused deep-dive into Dagger 2 dependency injection for Android. Each concept is implemented as a standalone, runnable example — making it easy to understand how each piece of Dagger works in isolation before combining them in production apps.

---

## Concepts covered

- `@Inject` — constructor, field, and method injection
- `@Module` — providing dependencies
- `@Component` — the bridge between providers and consumers
- `@Singleton` — application-scoped dependencies
- `@Scope` — custom scopes for Activity/Fragment lifecycle
- `@Subcomponent` — hierarchical component structure
- `@Binds` vs `@Provides` — when to use each
- `@Named` and `@Qualifier` — disambiguating multiple bindings
- Lazy and Provider injection

---

## Why Dagger 2?

Dagger 2 generates dependency injection code at compile time — no reflection, no runtime overhead. It's the standard DI framework for large-scale Android apps and the foundation for understanding Hilt (the modern successor).
